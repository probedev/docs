## Overview

The Probe.dev MCP (Model Context Protocol) server enables AI assistants to directly interact with the Probe.dev API and documentation. This integration allows you to:

- **Search documentation** - Find information across all Probe.dev docs
- **Analyze media files** - Run FFprobe, MediaInfo, and Probe Report analysis
- **Manage usage** - Check statistics and request logs
- **Authenticate requests** - Automatically handle API authentication

<Note>
You'll need a Probe.dev API key to use the MCP server. Get yours at [probe.dev/dashboard](https://www.probe.dev/dashboard).
</Note>

## Installation & Setup

### Step 1: Install the MCP Server

Run this command to install the Probe.dev MCP server:

```bash
npx mint-mcp add docs.probe.dev
```

When prompted, follow the authentication steps and select your preferred MCP clients.

### Step 2: Configure Your API Key

The MCP server needs your Probe.dev API key to make authenticated requests. Set it as an environment variable:

<Tabs>
<Tab title="macOS/Linux">
    ```bash
    export PROBE_API_TOKEN="your_api_key_here"
    ```
    
    To make it permanent, add this line to your shell profile:
    ```bash
    echo 'export PROBE_API_TOKEN="your_api_key_here"' >> ~/.zshrc
    source ~/.zshrc
    ```
</Tab>

<Tab title="Windows">
    ```powershell
    $env:PROBE_API_TOKEN="your_api_key_here"
    ```
    
    To make it permanent:
    ```powershell
    setx PROBE_API_TOKEN "your_api_key_here"
    ```
</Tab>
</Tabs>

<Warning>
Never commit your API key to version control. Always use environment variables or secure configuration files.
</Warning>

## Client-Specific Configuration

### Cursor Setup

1. **Open Cursor Settings**
   - Press `Cmd/Ctrl + ,` to open settings
   - Navigate to "Extensions" → "MCP"

2. **Add Probe.dev Server**
   ```json
   {
     "mcpServers": {
       "probe-dev": {
         "command": "npx",
         "args": ["mint-mcp", "run", "docs.probe.dev"],
         "env": {
           "PROBE_API_TOKEN": "${PROBE_API_TOKEN}"
         }
       }
     }
   }
   ```

3. **Restart Cursor** to activate the MCP server

### Claude Desktop Setup

1. **Locate Configuration File**
   - **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
   - **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

2. **Edit Configuration**
   ```json
   {
     "mcpServers": {
       "probe-dev": {
         "command": "npx",
         "args": ["mint-mcp", "run", "docs.probe.dev"],
         "env": {
           "PROBE_API_TOKEN": "${PROBE_API_TOKEN}"
         }
       }
     }
   }
   ```

3. **Restart Claude Desktop**

### Continue (VS Code) Setup

1. **Install Continue Extension** from the VS Code marketplace

2. **Configure MCP Server** in Continue's settings:
   ```json
   {
     "mcpServers": [
       {
         "name": "probe-dev",
         "command": "npx",
         "args": ["mint-mcp", "run", "docs.probe.dev"],
         "env": {
           "PROBE_API_TOKEN": "${PROBE_API_TOKEN}"
         }
       }
     ]
   }
   ```

## Troubleshooting

<AccordionGroup>
  <Accordion icon="exclamation-triangle" title="API Key Issues">
    **Symptoms**: Authentication errors, "401 Unauthorized" responses
    
    **Solutions**:
    - Verify your API key is correct at [probe.dev/dashboard](https://www.probe.dev/dashboard)
    - Check the environment variable is set: `echo $PROBE_API_TOKEN`
    - Restart your MCP client after setting the environment variable
    - Ensure no extra spaces or quotes in the API key
  </Accordion>
  
  <Accordion icon="plug" title="Connection Problems">
    **Symptoms**: "MCP server not responding", connection timeouts
    
    **Solutions**:
    - Check your internet connection
    - Verify the MCP server is installed: `npx mint-mcp list`
    - Try reinstalling: `npx mint-mcp remove docs.probe.dev && npx mint-mcp add docs.probe.dev`
    - Check for firewall/proxy issues blocking the connection
  </Accordion>
  
  <Accordion icon="gear" title="Configuration Issues">
    **Symptoms**: Tools not appearing, incorrect responses
    
    **Solutions**:
    - Validate your JSON configuration syntax
    - Ensure the `command` and `args` are correct
    - Check file permissions on configuration files
    - Restart your MCP client completely
  </Accordion>
  
  <Accordion icon="circle-question" title="Tool Not Working">
    **Symptoms**: Specific tools fail or return errors
    
    **Solutions**:
    - Check the [API status page](https://status.probe.dev)
    - Verify your account has sufficient quota
    - Test the API directly with cURL to isolate issues
    - Check request logs in your [dashboard](https://www.probe.dev/dashboard)
  </Accordion>
</AccordionGroup>

## Advanced Configuration

### Custom Tool Aliases

You can create custom aliases for frequently used operations:

```json
{
  "mcpServers": {
    "probe-dev": {
      "command": "npx",
      "args": ["mint-mcp", "run", "docs.probe.dev"],
      "env": {
        "PROBE_API_TOKEN": "${PROBE_API_TOKEN}"
      },
      "toolAliases": {
        "quick_analysis": "analyze_media_file",
        "docs": "search"
      }
    }
  }
}
```

### Rate Limiting Configuration

Configure request throttling to stay within API limits:

```json
{
  "mcpServers": {
    "probe-dev": {
      "command": "npx",
      "args": ["mint-mcp", "run", "docs.probe.dev"],
      "env": {
        "PROBE_API_TOKEN": "${PROBE_API_TOKEN}",
        "PROBE_RATE_LIMIT": "100"
      }
    }
  }
}
```
